## 背景

大语言模型（LLM）参数规模动辄数十亿甚至上千亿，全参数微调对显存的要求极高。LoRA（Low-Rank Adaptation）和 QLoRA 是两种在消费级硬件上实现高效微调的关键技术。

## LoRA 原理

LoRA 的核心思想是：**冻结预训练权重，只训练低秩矩阵**。假设原始权重矩阵为 W，LoRA 引入两个低秩矩阵 A 和 B，其中秩 r 远小于原始维度。

前向传播时，输出为原始输出加上低秩矩阵的输出：

```
h = Wx + BAx
```

## QLoRA 的改进

QLoRA 在 LoRA 基础上增加了 **4-bit 量化** 和 **双量化** 技术：

| 技术 | 显存节省 | 性能损失 |
|------|---------|---------|
| FP16 全参数 | 基准 | 无 |
| LoRA | ~50% | 极小 |
| QLoRA | ~90% | 轻微 |

## 实验配置

在 RTX 4090 (24GB) 上微调 Llama-2-7B：

```python
from peft import LoraConfig, get_peft_model
from transformers import AutoModelForCausalLM

model = AutoModelForCausalLM.from_pretrained(
    "meta-llama/Llama-2-7b-hf",
    load_in_4bit=True,  # QLoRA 关键
    device_map="auto",
)

lora_config = LoraConfig(
    r=16,
    lora_alpha=32,
    target_modules=["q_proj", "v_proj"],
    lora_dropout=0.05,
    bias="none",
    task_type="CAUSAL_LM",
)

model = get_peft_model(model, lora_config)
```

## 效果对比

- **LoRA**: 显存占用 ~14GB，训练速度较快
- **QLoRA**: 显存占用 ~8GB，适合更长序列

两种方法在下游任务上的性能差异通常在 1-2% 以内，QLoRA 的性价比极高。

## 总结

如果显存充足，LoRA 是更稳定的选择；如果硬件受限，QLoRA 几乎是最优解。
